# 发布与同步

这一页说明这个站点是如何从 `Obsidian` 文档同步并发布到 GitHub Pages 的。

## 源文档位置

- 源目录：`D:\obsidian\工作\一句话编程`
- 目标目录：`D:\code\ob_githubpage\docs\一句话编程`

## 同步步骤

在 `D:\code\ob_githubpage` 目录执行：

```powershell
.\scripts\sync_notes.ps1
```

这个脚本会做三件事：

1. 把源目录中的主文档和下位机文档复制到站点目录。
2. 保持 UTF-8 编码。
3. 将 `[[AI一句话编程]]` 这类 Obsidian 双链转换成网页可用的普通链接。

## 本地预览

```powershell
.\.venv\Scripts\Activate.ps1
mkdocs serve
```

本地预览地址默认是：

```text
http://127.0.0.1:8000
```

## 本地构建

```powershell
mkdocs build --strict
```

构建产物输出到：

```text
D:\code\ob_githubpage\site
```

## GitHub Pages 发布方式

仓库里已经包含 GitHub Pages 工作流文件：

```text
.github/workflows/deploy.yml
```

发布规则是：

1. 推送到 `main` 分支时自动触发。
2. GitHub Actions 构建 `MkDocs` 站点。
3. 构建结果自动部署到 GitHub Pages。

## 推荐工作流

1. 在 Obsidian 中更新源文档。
2. 运行 `.\scripts\sync_notes.ps1`。
3. 用 `mkdocs serve` 检查页面效果。
4. 提交并推送到 GitHub。
