# 发布与同步

这套站点现在按 `D:\obsidian\工作` 的一级目录自动分项目发布。也就是说：

- `D:\obsidian\工作\一句话编程` 会变成一个项目
- `D:\obsidian\工作\导盲兔` 会变成一个项目
- 后续新增任意一级目录，重新同步后也会自动出现在站点里

## 现在只需要维护一个地方

后续所有内容都建议只在 `D:\obsidian\工作` 里维护。

### 原始文档放哪里

直接放在项目目录里，例如：

- `D:\obsidian\工作\一句话编程\AI一句话编程.md`
- `D:\obsidian\工作\一句话编程\下位机文档\按键开关模块下位机开发文档.md`

这些内容会自动发布到站点的 `文档/` 区域。

### 补充页放哪里

如果你想写“协议速览”“模块索引”“阅读导航”“总结页”这类补充页，请放到项目里的 `补充页` 目录，例如：

- `D:\obsidian\工作\一句话编程\补充页\协议速览.md`
- `D:\obsidian\工作\一句话编程\补充页\模块索引.md`

这些内容会自动发布到站点的 `补充页/` 区域，并自动进入左侧导航。

## 源文档位置

- 工作目录：`D:\obsidian\工作`
- 站点项目目录：`D:\code\ob_githubpage\docs\项目`
- 同步脚本：`D:\code\ob_githubpage\scripts\sync_notes.ps1`

## 同步脚本会做什么

执行脚本后，会自动完成这些事情：

1. 扫描 `D:\obsidian\工作` 下的一级文件夹，把每个文件夹视为一个项目。
2. 把项目里的普通文档发布到 `docs/项目/<项目名>/文档/`。
3. 把项目里的 `补充页/` 发布到 `docs/项目/<项目名>/补充页/`。
4. 自动生成 `docs/项目/index.md` 项目总览页。
5. 自动生成每个项目的 `index.md` 项目主页。
6. 自动生成左侧树状导航。
7. 把 Obsidian 里的 `[[双链]]` 转成站点可访问的普通 Markdown 链接。

## 手动同步

在 `D:\code\ob_githubpage` 目录执行：

```powershell
.\scripts\sync_notes.ps1
```

## 本地预览

```powershell
.\.venv\Scripts\Activate.ps1
mkdocs serve
```

默认预览地址：

```text
http://127.0.0.1:8000
```

## 本地构建检查

```powershell
mkdocs build --strict
```

构建产物输出到：

```text
D:\code\ob_githubpage\site
```

## GitHub Pages 发布方式

仓库已经包含 GitHub Pages 工作流：

```text
.github/workflows/deploy.yml
```

发布规则是：

1. 提交并推送到 `main` 分支。
2. GitHub Actions 自动构建 MkDocs 站点。
3. 构建结果自动部署到 GitHub Pages。

## 日常更新建议

如果你只是修改了 Obsidian 里的内容，直接使用插件里的一键发布就够了。它会依次帮你做：

1. 执行同步脚本。
2. 执行 `mkdocs build --strict`。
3. 检查 Git 改动并自动提交。
4. 推送到 GitHub。
