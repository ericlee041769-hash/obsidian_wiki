# 模块索引

这一页把首期模组放在同一个视角下对比，方便做排期、拆任务和评估联调优先级。

## 首期模组概览

| 模组 | 类型 | 主要能力 | 风险等级 | 开发优先级 | 文档 |
| ---- | ---- | ---- | ---- | ---- | ---- |
| 按键开关模块 | 输入 | `read.state` `subscribe.click` `subscribe.long_press` | 低 | 高 | [查看](./文档/下位机文档/按键开关模块下位机开发文档.md) |
| 三色LED模块 | 输出 | `write.on` `write.rgb` `write.brightness` `execute.blink` | 低 | 高 | [查看](./文档/下位机文档/三色LED模块下位机开发文档.md) |
| 土壤湿度模块 | 传感器 | `read.raw` `read.percent` `subscribe.threshold` | 低 | 高 | [查看](./文档/下位机文档/土壤湿度模块下位机开发文档.md) |
| 温湿度模块 | 传感器 | `read.temperature` `read.humidity` `subscribe.temperature_threshold` | 低 | 中高 | [查看](./文档/下位机文档/温湿度模块下位机开发文档.md) |
| PWM舵机模块 | 执行器 | `write.angle` `execute.home` `execute.stop` | 中 | 中 | [查看](./文档/下位机文档/PWM舵机模块下位机开发文档.md) |

## 建议开发顺序

1. 先完成按键开关模块和三色 LED 模块，最快形成“输入 -> 输出”的演示闭环。
2. 再补土壤湿度和温湿度模块，把传感器读取、阈值事件、配置持久化跑通。
3. 最后接入 PWM 舵机模块，把执行器边界保护和频率限制加上。

## 推荐联调组合

| 组合 | 作用 |
| ---- | ---- |
| 按键开关 + 三色LED | 最快验证事件订阅、写入控制、状态读取 |
| 土壤湿度 + 三色LED | 验证阈值触发与联动反馈 |
| 温湿度 + 三色LED | 验证双通道传感器与阈值恢复事件 |
| 按键开关 + PWM舵机 | 验证简单执行链路与安全限制 |

## 当前每类模组的关注重点

### 输入模组

- 核心风险在于去抖、事件重复上报和时间语义不稳定。

### 输出模组

- 核心风险在于参数边界、写入吞吐和状态一致性。

### 传感器模组

- 核心风险在于采样稳定性、异常值处理和阈值抖动。

### 执行器模组

- 核心风险在于动作边界、执行频率和上电默认行为。
