# 1. 文档目标

本文档用于定义首期 `按键开关模块` 的下位机开发范围、功能要求、协议行为、状态机、测试项和验收标准，作为固件开发、联调和测试的直接依据。

本文档默认继承主文档 [AI一句话编程](../AI一句话编程.md) 中的整体架构、自动编号、心跳机制和 CANFD 协议 v1 约束。

# 2. 模块定位

按键开关模块属于首期优先实现的低风险交互输入模组，用于：

1. 提供最基础的人机输入能力。
2. 支持项目脚本通过事件订阅方式接收按下、松开、单击和长按事件。
3. 支持本地按钮作为灯光、蜂鸣器、舵机等模块的低门槛触发源。
4. 作为系统最简单、最稳定的教学示范模组之一。

该模块本身不执行高风险动作，但它会成为大量自动化脚本的触发入口，因此要求事件语义稳定、去抖可靠、时间戳一致。

# 3. 首期实现范围

## 3.1 必须实现

1. 模组上电 `Hello` 自报。
2. 自动编号与重编号。
3. 心跳保活。
4. `Discover` 和 `GetCapabilities`。
5. `read.state`。
6. `read.status`。
7. `read.press_count`。
8. `subscribe.press`。
9. `subscribe.release`。
10. `subscribe.click`。
11. `subscribe.long_press`。
12. `configure.debounce_ms`。
13. `configure.long_press_ms`。
14. `configure.active_level`。
15. 本地参数持久化。
16. 两个按钮调整 `preferred_slot`。

## 3.2 首期可选实现

1. 双击事件。
2. 板载状态 LED。
3. 多按键版本复用同一协议。
4. OTA。

## 3.3 首期不做

1. 矩阵键盘扫描。
2. 本地宏录制。
3. 多击组合手势。
4. 离线保存完整输入历史。

# 4. 硬件假设

本文档基于以下硬件假设编写，若后续选型不同，需要同步修正文档和参数范围。

## 4.1 模块组成

1. MCU 一颗。
2. CANFD 收发器一颗。
3. 用户功能按键一个。
4. `slot_up` / `slot_down` 按钮各一个。
5. 状态 LED 一颗。
6. 非易失存储区，用于保存 `module_uid`、参数和地址偏好。

## 4.2 电气假设

1. 模块逻辑电压为 `3.3V`。
2. 功能按键默认为数字输入模式。
3. 输入脚支持上拉或下拉配置。
4. 首期默认按键输入频率远低于总线带宽上限。

## 4.3 地址与身份

1. 每个模块出厂必须烧录唯一 `module_uid`。
2. 首次上电默认 `assigned_slot = 0xFF`。
3. `preferred_slot` 默认为 `1`，真正生效的地址由主机分配。

# 5. 功能需求

## 5.1 读取功能

模块必须提供以下读取能力：

1. `read.state`
2. `read.status`
3. `read.press_count`

说明：

1. `read.state` 返回当前按键实时状态。
2. `read.status` 返回当前模块状态位图。
3. `read.press_count` 返回上电以来有效按压计数。

## 5.2 配置功能

模块必须支持以下配置能力：

1. `configure.debounce_ms`
2. `configure.long_press_ms`
3. `configure.active_level`

## 5.3 事件功能

模块必须支持：

1. `subscribe.press`
2. `subscribe.release`
3. `subscribe.click`
4. `subscribe.long_press`

说明：

1. 单次按下后只允许产生一个 `press` 事件。
2. 单次松开后只允许产生一个 `release` 事件。
3. 在满足点击时间窗时产生 `click` 事件。
4. 在持续按下超过阈值后产生 `long_press` 事件。

## 5.4 地址管理功能

模块必须支持：

1. 未分配状态接入。
2. 接收主机下发的 `assigned_slot`。
3. 保存 `preferred_slot`。
4. 按钮调整 `preferred_slot` 并发起改号请求。
5. 心跳中携带当前地址信息。

# 6. 模块能力定义

模块类型编码沿用主文档，`button = 0x01`。

## 6.1 能力表

| 能力 ID | 类型 | 数据类型 | 必须实现 | 说明 |

| `read.status` | Read | BitMap | 是 | 当前状态位 |
| `read.press_count` | Read | Int32 | 是 | 上电后累计有效按压次数 |
| `configure.debounce_ms` | Configure | Int32 | 是 | 配置去抖时间 |
| `configure.long_press_ms` | Configure | Int32 | 是 | 配置长按判定时间 |
| `configure.active_level` | Configure | Enum | 是 | 高电平有效或低电平有效 |
| `subscribe.press` | Subscribe | Enum | 是 | 订阅按下事件 |
| `subscribe.release` | Subscribe | Enum | 是 | 订阅松开事件 |
| `subscribe.click` | Subscribe | Enum | 是 | 订阅单击事件 |
| `subscribe.long_press` | Subscribe | Enum | 是 | 订阅长按事件 |

# 7. 数据定义

## 7.1 按键状态

`read.state` 建议返回枚举值：

| 值 | 含义 |
| ---- | ---- |
| `0` | released |
| `1` | pressed |

## 7.2 状态位建议

`read.status` 建议返回位图：

| Bit | 含义 |
| ---- | ---- |
| `bit0` | 在线且已分配地址 |
| `bit1` | 去抖参数有效 |
| `bit2` | 正处于按下状态 |
| `bit3` | 长按判定已触发 |
| `bit4` | 最近一次配置写入成功 |
| `bit5` | 地址等待重新分配 |

## 7.3 事件 Payload 建议

按键事件建议最少包含：

1. `event_type`
2. `event_ts_ms`
3. `press_duration_ms`
4. `press_count`

# 8. 参数范围

## 8.1 默认参数

| 参数 | 默认值 | 说明 |
| ---- | ---- | ---- |
| `debounce_ms` | `30` | 默认去抖时间 |
| `long_press_ms` | `800` | 默认长按阈值 |
| `active_level` | `0` | 默认低电平有效 |
| `heartbeat_interval_sec` | `10` | 默认心跳周期 |
| `lease_timeout_sec` | `30` | 默认离线租约 |

## 8.2 允许范围

| 参数 | 最小值 | 最大值 |
| ---- | ---- | ---- |
| `debounce_ms` | `5` | `100` |
| `long_press_ms` | `300` | `5000` |
| `active_level` | `0` | `1` |

# 9. 协议行为

## 9.1 上电流程

模块上电后必须按以下顺序执行：

1. 初始化时钟、GPIO、CANFD、定时器和存储。
2. 读取本地持久化参数。
3. 若无 `module_uid`，则判定为制造错误，不进入正常运行。
4. 进入 `Unassigned` 或 `Assigned` 启动状态。
5. 发送 `Hello`。
6. 等待主机 `Discover`、`GetCapabilities` 和地址配置。

## 9.2 Hello Payload

建议 `Hello` 至少包含：

```yaml
module_uid: 2A-11-9C-00-00-00-11
module_type: button
preferred_slot: 1
assigned_slot: 255
address_state: unassigned
fw_version: 1.0.0
protocol_version: 1
button_count: 1
```

## 9.3 Discover 响应

模块响应 `Discover` 时，建议返回：

1. `module_uid`
2. `module_type`
3. `preferred_slot`
4. `assigned_slot`
5. `fw_version`
6. `protocol_version`
7. `button_count`
8. `manufacturer`
9. `model`

## 9.4 Read 响应

### `read.state`

返回：

1. 编码类型 `Enum`
2. 当前按键状态

### `read.status`

返回：

1. 编码类型 `BitMap`
2. 当前状态位图

### `read.press_count`

返回：

1. 编码类型 `Int32`
2. 当前累计按压次数

## 9.5 Configure 写入

模块收到 `Configure` 时必须：

1. 检查子功能码是否合法。
2. 检查参数类型是否合法。
3. 检查参数值是否越界。
4. 更新运行时配置。
5. 对持久化参数做异步写入。
6. 返回 ACK 或错误码。

## 9.6 Subscribe 行为

模块收到按键事件订阅后：

1. 保存订阅状态。
2. 仅在状态切换时上报对应事件。
3. 不允许在抖动期间重复上报。

# 10. 自动编号与按钮改号

## 10.1 地址模型

1. 同类型模块内 `assigned_slot` 唯一。
2. `0xFF` 表示未分配。
3. 主机是唯一地址分配者。
4. 模块只提出 `preferred_slot`，不能自行宣布地址生效。

## 10.2 按钮行为

### `slot_up`

1. 短按：`preferred_slot + 1`
2. 达到最大值后首期建议停留在最大值

### `slot_down`

1. 短按：`preferred_slot - 1`
2. 到达最小值后停留在 `1`

## 10.3 按钮改号后的动作

1. 更新本地 `preferred_slot`
2. 发送 `Event.SlotChangeRequest`
3. 等待主机下发新的 `assigned_slot`
4. 收到确认后更新本地状态

# 11. 心跳与在线状态

## 11.1 心跳内容

建议心跳至少包含：

```yaml
module_uid: 2A-11-9C-00-00-00-11
module_type: button
assigned_slot: 1
address_state: assigned
latest_state: released
press_count: 25
status_flags: 0
uptime_sec: 180
```

## 11.2 心跳规则

1. 默认每 `10 秒` 发送一次。
2. 若主机要求，也可配置为 `30 秒` 一次。
3. 心跳发送失败时可记录错误计数，但不允许自行改地址。

## 11.3 主机离线判定配合

模块侧只负责按周期发送心跳，离线判定由主机负责。

# 12. 去抖与事件设计

## 12.1 去抖策略

建议采用固定时间窗去抖：

1. 采集到电平变化后启动去抖计时。
2. 在 `debounce_ms` 时间窗内持续稳定才认定为有效变化。
3. 去抖成功后再更新状态机。

## 12.2 点击判定

1. 按下后未超过 `long_press_ms` 且正常松开，判定为单击。
2. 单次按压只允许产生一个 `click` 事件。

## 12.3 长按判定

1. 持续按下时间超过 `long_press_ms` 后触发 `long_press`。
2. 长按触发后直到松开前不得重复上报。

# 13. 事件设计

## 13.1 `press`

按键从释放变为按下时产生。

## 13.2 `release`

按键从按下变为释放时产生。

## 13.3 `click`

按键完成一次有效短按时产生。

## 13.4 `long_press`

按键持续按下超过阈值时产生。

# 14. 固件状态机

建议固件至少包含以下状态：

1. `BOOT`
2. `UNASSIGNED`
3. `ASSIGN_PENDING`
4. `ASSIGNED_IDLE`
5. `PRESSED`
6. `FAULT`
7. `OTA`

## 14.1 状态说明

### `BOOT`

初始化硬件、读取参数、准备上报。

### `UNASSIGNED`

尚未获得主机分配的逻辑号。

### `ASSIGN_PENDING`

已经收到地址分配，等待确认写入和切换。

### `ASSIGNED_IDLE`

已分配地址，等待输入变化或协议请求。

### `PRESSED`

按键当前处于有效按下状态。

### `FAULT`

存在严重错误，例如 GPIO 初始化失败、存储损坏或配置非法。

# 15. 本地持久化内容

模块建议持久化以下参数：

1. `module_uid`
2. `preferred_slot`
3. 上次 `assigned_slot`
4. `debounce_ms`
5. `long_press_ms`
6. `active_level`

说明：

1. `assigned_slot` 仅作为恢复参考。
2. 配置写入 Flash 时要做磨损控制。

# 16. 错误处理

## 16.1 推荐内部错误码

| 错误码 | 含义 |
| ---- | ---- |
| `0x01` | GPIO 初始化失败 |
| `0x02` | 按键输入异常 |
| `0x03` | 参数非法 |
| `0x04` | Flash 写入失败 |
| `0x05` | 地址未分配 |

## 16.2 异常处理原则

1. 参数非法时拒绝写入。
2. 关键初始化失败时进入 `FAULT`。
3. 事件异常时允许继续上报心跳，便于定位问题。
4. 不允许因为一次抖动异常就重启整机。

# 18. 测试项

## 18.1 功能测试

1. 上电后是否能正确发送 `Hello`
2. 主机分号后是否能进入 `Assigned`
3. `read.state` 是否反映真实状态
4. `read.press_count` 是否正确累计
5. 配置写入后是否立即生效
6. 掉电重启后参数是否保留

## 18.2 事件测试

1. 按下是否产生 `press`
2. 松开是否产生 `release`
3. 短按是否产生一次 `click`
4. 长按是否产生一次 `long_press`
5. 抖动时是否避免重复事件

## 18.3 地址与心跳测试

1. 首次插入是否自动分号
2. 同类型两个模块同时插入是否能正确区分
3. 按钮改号后是否能正确触发重分发
4. 心跳是否按设定周期发送
5. 心跳超时后主机是否能判定离线

## 18.4 异常测试

1. 去抖参数越界时是否拒绝生效
2. GPIO 异常时状态位是否正确
3. Flash 写入失败时是否能返回错误

# 19. 验收标准

满足以下条件即可认为首期按键开关模块下位机开发完成：

1. 能完成 `Hello -> Discover -> GetCapabilities -> AssignSlot -> Heartbeat` 全流程。
2. 能稳定返回 `read.state` 和 `read.press_count`。
3. 能正确上报 `press`、`release`、`click`、`long_press`。
4. 能正确保存和加载去抖及长按参数。
5. 连续运行 `24 小时` 无异常死机或协议失联。
