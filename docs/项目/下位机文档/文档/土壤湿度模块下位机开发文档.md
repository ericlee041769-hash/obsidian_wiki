# 1. 文档目标

本文档用于定义首期 `土壤湿度模块` 的下位机开发范围、功能要求、协议行为、状态机、测试项和验收标准，作为固件开发、联调和测试的直接依据。

本文档默认继承主文档 [[AI一句话编程]]中的整体架构、自动编号、心跳机制和 CANFD 协议 v1 约束。

# 2. 模块定位

土壤湿度模块属于首期优先实现的低风险环境感知模组，用于：

1. 读取土壤湿度原始值。
2. 将原始值转换为标准化百分比。
3. 支持云端项目脚本通过 `send_cmd("soil_moisture", slot, "read.percent", nil)` 读取当前湿度。
4. 支持阈值订阅，在湿度过低或恢复到安全范围时主动上报事件。

该模块本身不直接驱动高风险执行器，但它会作为自动浇水、提醒告警、联动灯光或蜂鸣器的核心输入源。

# 3. 首期实现范围

## 3.1 必须实现

1. 模组上电 `Hello` 自报。
2. 自动编号与重编号。
3. 心跳保活。
4. `Discover` 和 `GetCapabilities`。
5. `read.raw`。
6. `read.percent`。
7. `read.status`。
8. `configure.sample_interval_ms`。
9. `configure.calibration_dry`。
10. `configure.calibration_wet`。
11. `subscribe.threshold`。
12. 阈值事件主动上报。
13. 本地参数持久化。
14. 两个按钮调整 `preferred_slot`。

## 3.2 首期可选实现

1. 板载状态 LED。
2. 传感器探头断开检测。
3. 温度补偿。
4. 本地校准模式。
5. OTA。

## 3.3 首期不做

1. 多探头同时接入。
2. 自动识别土壤类型。
3. 本地复杂控制逻辑。
4. 脱机保存长期历史曲线。

# 4. 硬件假设

本文档基于以下硬件假设编写，若后续选型不同，需要同步修正文档和参数范围。

## 4.1 模块组成

1. MCU 一颗。
2. CANFD 收发器一颗。
3. 土壤湿度传感探头一路。
4. ADC 采样输入一路。
5. 状态 LED 一颗。
6. `slot_up` / `slot_down` 按钮各一个。
7. 非易失存储区，用于保存 `module_uid`、校准参数和上次地址偏好。

## 4.2 电气假设

1. 模块逻辑电压为 `3.3V`。
2. 探头输出接入 MCU ADC 通道。
3. ADC 分辨率默认为 `12 bit`，原始范围 `0 ~ 4095`。
4. 推荐使用电容式土壤湿度探头，优先避免电阻式探头长期腐蚀问题。

## 4.3 地址与身份

1. 每个模块出厂必须烧录唯一 `module_uid`。
2. 首次上电默认 `assigned_slot = 0xFF`。
3. `preferred_slot` 默认为 `1`，但真正生效的地址由主机分配。

# 5. 功能需求

## 5.1 读取功能

模块必须提供以下读取能力：

1. `read.raw`
2. `read.percent`
3. `read.status`

说明：

1. `read.raw` 返回 ADC 过滤后的原始值。
2. `read.percent` 返回 `0 ~ 100` 的土壤湿度百分比。
3. `read.status` 返回当前模块状态位图，例如在线、校准有效、探头异常、阈值触发中。

## 5.2 配置功能

模块必须支持以下配置能力：

1. `configure.sample_interval_ms`
2. `configure.calibration_dry`
3. `configure.calibration_wet`
4. `configure.threshold_percent`
5. `configure.threshold_hysteresis`

## 5.3 事件功能

模块必须支持：

1. `subscribe.threshold`
2. 当湿度低于阈值时主动上报。
3. 当湿度恢复到阈值加回差以上时主动上报恢复事件。

## 5.4 地址管理功能

模块必须支持：

1. 未分配状态接入。
2. 接收主机下发的 `assigned_slot`。
3. 保存 `preferred_slot`。
4. 按钮调整 `preferred_slot` 并发起改号请求。
5. 心跳中携带当前地址信息。

# 6. 模块能力定义

模块类型编码沿用主文档，`soil_moisture = 0x08`。

## 6.1 能力表

| 能力 ID | 类型 | 数据类型 | 必须实现 | 说明 |
| ---- | ---- | ---- | ---- | ---- |
| `read.raw` | Read | Int32 | 是 | 读取原始 ADC 值 |
| `read.percent` | Read | Int32 | 是 | 读取湿度百分比，范围 `0 ~ 100` |
| `read.status` | Read | BitMap | 是 | 读取状态位 |
| `configure.sample_interval_ms` | Configure | Int32 | 是 | 配置采样周期 |
| `configure.calibration_dry` | Configure | Int32 | 是 | 配置干燥校准值 |
| `configure.calibration_wet` | Configure | Int32 | 是 | 配置湿润校准值 |
| `configure.threshold_percent` | Configure | Int32 | 是 | 配置阈值 |
| `configure.threshold_hysteresis` | Configure | Int32 | 是 | 配置回差 |
| `subscribe.threshold` | Subscribe | Int32 | 是 | 订阅阈值事件 |

## 6.2 推荐子功能码

| 子功能码 | 能力 ID |
| ---- | ---- |
| `0x01` | `read.raw` |
| `0x02` | `read.percent` |
| `0x03` | `read.status` |
| `0x11` | `configure.sample_interval_ms` |
| `0x12` | `configure.calibration_dry` |
| `0x13` | `configure.calibration_wet` |
| `0x14` | `configure.threshold_percent` |
| `0x15` | `configure.threshold_hysteresis` |
| `0x21` | `subscribe.threshold` |

# 7. 数据定义

## 7.1 原始值

1. 数据源为 ADC 采样值。
2. 默认范围 `0 ~ 4095`。
3. 原始值需在滤波后再对外返回。

## 7.2 百分比换算

默认采用线性换算：

```text
percent = clamp((dry_raw - current_raw) * 100 / (dry_raw - wet_raw), 0, 100)
```

说明：

1. `dry_raw` 表示空气中或干土时的校准值。
2. `wet_raw` 表示湿润土壤或水中时的校准值。
3. 若 `dry_raw <= wet_raw`，视为校准非法，模块进入配置错误状态。

## 7.3 状态位建议

`read.status` 建议返回位图：

| Bit | 含义 |
| ---- | ---- |
| `bit0` | 在线且已分配地址 |
| `bit1` | 校准参数有效 |
| `bit2` | 传感器值异常 |
| `bit3` | 阈值低湿告警中 |
| `bit4` | 最近一次配置写入成功 |
| `bit5` | 地址等待重新分配 |

| `calibration_dry` | `3000` | 默认干燥校准值 |
| `calibration_wet` | `1200` | 默认湿润校准值 |
| `threshold_percent` | `35` | 默认低湿阈值 |
| `threshold_hysteresis` | `3` | 默认恢复回差 |
| `heartbeat_interval_sec` | `10` | 默认心跳周期 |
| `lease_timeout_sec` | `30` | 默认离线租约 |

## 8.2 允许范围

| 参数 | 最小值 | 最大值 |
| ---- | ---- | ---- |
| `sample_interval_ms` | `1000` | `60000` |
| `calibration_dry` | `0` | `4095` |
| `calibration_wet` | `0` | `4095` |
| `threshold_percent` | `0` | `100` |
| `threshold_hysteresis` | `1` | `20` |

# 9. 协议行为

## 9.1 上电流程

模块上电后必须按以下顺序执行：

1. 初始化时钟、GPIO、ADC、CANFD、定时器、存储。
2. 读取本地持久化参数。
3. 若本地无 `module_uid`，则判定为制造错误，不进入正常运行。
4. 进入 `Unassigned` 或 `Assigned` 启动状态。
5. 发送 `Hello`。
6. 等待主机 `Discover`、`GetCapabilities` 和地址配置。

## 9.2 Hello Payload

建议 `Hello` 至少包含：

```yaml
module_uid: 2A-11-9C-00-00-00-02
module_type: soil_moisture
preferred_slot: 1
assigned_slot: 255
address_state: unassigned
fw_version: 1.0.0
protocol_version: 1
```

## 9.3 Discover 响应

模块响应 `Discover` 时，建议返回：

1. `module_uid`
2. `module_type`
3. `preferred_slot`
4. `assigned_slot`
5. `fw_version`
6. `protocol_version`
7. `manufacturer`
8. `model`

## 9.4 GetCapabilities 响应

必须返回本模块支持的能力清单、数据类型、范围和默认参数。

## 9.5 Read 响应

### `read.raw`

返回：

1. 编码类型 `Int32`
2. 当前过滤后的 ADC 原始值

### `read.percent`

返回：

1. 编码类型 `Int32`
2. 当前湿度百分比

### `read.status`

返回：

1. 编码类型 `BitMap`
2. 当前状态位图

## 9.6 Configure 写入

模块收到 `Configure` 时必须：

1. 检查子功能码是否合法。
2. 检查参数类型是否合法。
3. 检查参数值是否越界。
4. 写入 RAM。
5. 若为持久化参数，则异步写入 Flash。
6. 返回 ACK 或错误码。

## 9.7 Subscribe 行为

模块收到 `subscribe.threshold` 后：

1. 保存阈值订阅状态。
2. 在采样任务中检测阈值穿越。
3. 仅在状态变化时主动上报事件。

# 10. 自动编号与按钮改号

## 10.1 地址模型

1. 同类型模组内 `assigned_slot` 唯一。
2. `0xFF` 表示未分配。
3. 主机是唯一地址分配者。
4. 模块只能提出 `preferred_slot`，不能自行宣布地址生效。

## 10.2 按钮行为

### `slot_up`

1. 短按：`preferred_slot + 1`
2. 达到最大值后可选择回绕到 `1`，也可以停在最大值，建议首期停在最大值

### `slot_down`

1. 短按：`preferred_slot - 1`
2. 到达最小值后建议停在 `1`

## 10.3 按钮改号后的动作

1. 更新本地 `preferred_slot`
2. 发送 `Event.SlotChangeRequest`
3. 等待主机下发新的 `assigned_slot`
4. 收到确认后更新本地状态

## 10.4 本地显示建议

若有 LED，可用以下方式反馈：

1. 未分配：慢闪
2. 正常在线：常亮或低频闪烁
3. 等待改号确认：快闪
4. 配置错误：双闪

# 11. 心跳与在线状态

## 11.1 心跳内容

建议心跳至少包含：

```yaml
module_uid: 2A-11-9C-00-00-00-02
module_type: soil_moisture
assigned_slot: 1
address_state: assigned
latest_percent: 42
status_flags: 0
uptime_sec: 180
```

## 11.2 心跳规则

1. 默认每 `10 秒` 发送一次。
2. 若主机要求，也可配置为 `30 秒`。
3. 心跳发送失败时，模块本地可记录错误计数，但不自行改地址。

## 11.3 主机离线判定配合

模块侧只负责按周期发送心跳，离线判断由主机负责。模块无需自行声明“我已经离线”。

# 12. 采样与滤波设计

## 12.1 采样任务

建议采用独立周期任务：

1. 每 `sample_interval_ms` 触发一次。
2. 单次采样读取 `N` 次 ADC，建议 `N = 8`。
3. 去掉最大值和最小值各一个。
4. 对剩余值取平均。
5. 生成 `raw_value`。
6. 由 `raw_value` 计算 `percent_value`。

## 12.2 平滑策略

为减少抖动，建议增加一阶低通或滑动平均：

```text
filtered = alpha * current + (1 - alpha) * previous
```

首期可选：

1. 若资源紧张，仅保留多次采样均值。
2. 若资源允许，增加滑动平均窗口 `3 ~ 5`。

## 12.3 异常值处理

若出现以下情况，应标记异常：

1. ADC 连续返回固定极值。
2. 原始值超出合理区间。
3. 校准参数非法。

异常时：

1. `read.status` 标记异常位。
2. `read.percent` 可返回上次有效值，或返回约定错误响应。
3. 记录错误日志计数。

# 13. 阈值事件设计

## 13.1 触发条件

若已订阅阈值事件：

1. 当 `percent <= threshold_percent` 且之前不在告警状态时，触发“低湿事件”。
2. 当 `percent >= threshold_percent + threshold_hysteresis` 且之前在告警状态时，触发“恢复事件”。

## 13.2 为什么要有回差

没有回差时，湿度值在阈值附近波动会导致频繁来回上报。

## 13.3 事件 Payload 建议

```yaml
module_uid: 2A-11-9C-00-00-00-02
assigned_slot: 1
event: threshold_crossed
current_percent: 28
threshold_percent: 35
direction: below
```

# 14. 固件状态机

建议固件至少包含以下状态：

1. `BOOT`
2. `UNASSIGNED`
3. `ASSIGN_PENDING`
4. `ASSIGNED_IDLE`
5. `MEASURING`
6. `FAULT`
7. `OTA`

## 14.1 状态说明

### `BOOT`

初始化硬件、读取参数、准备上报。

### `UNASSIGNED`

尚未获得主机分配的逻辑号，可接收 `AssignSlot`。

### `ASSIGN_PENDING`

已经收到地址分配，等待确认写入和切换。

### `ASSIGNED_IDLE`

已分配地址，等待采样或协议请求。

### `MEASURING`

正在执行周期采样与阈值判断。

### `FAULT`

存在严重错误，例如存储损坏、校准非法、ADC 初始化失败。

# 15. 本地持久化内容

模块建议持久化以下参数：

1. `module_uid`
2. `preferred_slot`
3. 上次 `assigned_slot`
4. `calibration_dry`
5. `calibration_wet`
6. `sample_interval_ms`
7. `threshold_percent`
8. `threshold_hysteresis`

说明：

1. `assigned_slot` 只是恢复参考，最终仍以主机分配为准。
2. 参数写 Flash 时要做磨损控制。

# 16. 错误处理

## 16.1 推荐内部错误码

| 错误码 | 含义 |
| ---- | ---- |
| `0x01` | ADC 初始化失败 |
| `0x02` | ADC 采样异常 |
| `0x03` | 校准参数非法 |
| `0x04` | 参数越界 |
| `0x05` | Flash 写入失败 |
| `0x06` | 地址未分配 |

## 16.2 异常处理原则

1. 参数非法时拒绝写入。
2. 关键初始化失败时进入 `FAULT`。
3. 心跳仍可继续发送，便于主机定位故障。
4. 不允许因为一次采样失败就复位整机。

# 18. 测试项

## 18.1 功能测试

1. 上电后是否能正确发送 `Hello`
2. 主机分号后是否能进入 `Assigned`
3. `read.raw` 是否返回有效值
4. `read.percent` 是否在 `0 ~ 100`
5. 配置写入后是否立即生效
6. 掉电重启后参数是否保留

## 18.2 地址与心跳测试

1. 首次插入是否自动分号
2. 同类型两个模块同时插入是否能正确区分
3. 按钮改号后是否能正确触发重分发
4. 心跳是否按设定周期发送
5. 心跳超时后主机是否能判定离线

## 18.3 阈值测试

1. 湿度降低到阈值以下时是否主动上报
2. 恢复到阈值加回差以上时是否主动上报恢复
3. 阈值附近抖动时是否避免频繁重复事件

## 18.4 异常测试

1. 校准参数颠倒时是否拒绝生效
2. ADC 异常时状态位是否正确
3. Flash 写入失败时是否能返回错误

# 19. 验收标准

满足以下条件即可认为首期土壤湿度模块下位机开发完成：

1. 能完成 `Hello -> Discover -> GetCapabilities -> AssignSlot -> Heartbeat` 全流程。
2. 能稳定返回 `read.raw` 和 `read.percent`。
3. 能正确保存和加载校准参数。
4. 能通过按钮触发改号请求。
5. 能在阈值穿越时主动上报事件。
6. 连续运行 `24 小时` 无异常死机或协议失联。
