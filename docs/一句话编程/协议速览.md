# 协议速览

这一页不是完整协议，而是把最值得先掌握的部分压缩成一页，方便开发和联调时快速回看。

## 协议 v1 想解决什么

根据 [AI一句话编程](./AI一句话编程.md)，协议 v1 重点覆盖：

- 设备发现
- 能力查询
- 数据读取
- 动作写入
- 事件上报
- 错误反馈
- 分片传输
- OTA 预留

## 关键对象

| 名称 | 作用 |
| ---- | ---- |
| `module_uid` | 设备出厂唯一标识，用于稳定识别物理模组 |
| `preferred_slot` | 模组偏好的逻辑号 |
| `assigned_slot` | 主机实际分配的逻辑号 |
| `inventory_revision` | 当前设备映射版本号，地址变化时必须递增 |

## 典型链路

### 设备接入

1. 模组上电发送 `Hello`
2. 主机发 `Discover`
3. 模组返回基础信息
4. 主机发 `GetCapabilities`
5. 模组返回能力清单
6. 主机分配 `assigned_slot`
7. 模组开始按周期发送 `Heartbeat`

### 事件联动

1. 主机通过 `Subscribe` 注册阈值或事件
2. 模组本地判断是否触发
3. 模组通过 `Event` 主动上报

### OTA 预留流程

1. 主机发送 `OtaBegin`
2. 模组检查升级条件
3. 主机分片发送 `OtaChunk`
4. 模组逐片校验并回 ACK
5. 主机发送 `OtaCommit`
6. 模组重启并重新 `Hello`

## 帧类型

| 帧类型 | 名称 | 作用 |
| ---- | ---- | ---- |
| `0x01` | Request | 主机主动请求 |
| `0x02` | Response | 模组响应 |
| `0x03` | Event | 模组主动事件上报 |
| `0x04` | Heartbeat | 在线状态与心跳 |
| `0x05` | Fragment | 分片传输 |
| `0x06` | OtaCtrl | OTA 控制 |

## 常用操作码

| 操作码 | 名称 | 用途 |
| ---- | ---- | ---- |
| `0x01` | Hello | 上电自报 |
| `0x02` | Discover | 查询基础信息 |
| `0x03` | GetCapabilities | 查询能力清单 |
| `0x04` | Read | 读取能力值 |
| `0x05` | Write | 写入能力值 |
| `0x06` | Execute | 执行动作 |
| `0x07` | Configure | 下发参数 |
| `0x08` | Subscribe | 订阅事件 |
| `0x0A` | OtaBegin | 开始升级 |
| `0x0B` | OtaChunk | 升级分片 |
| `0x0C` | OtaCommit | 提交升级 |

## 地址管理最值得记住的规则

1. `assigned_slot = 0xFF` 表示未分配。
2. 逻辑号作用域限定在 `module_type` 内，不要求全局唯一。
3. 每次地址变化都必须递增 `inventory_revision`。
4. 地址变化后，主机需要重绑或重启受影响项目。

## 调试时优先看哪些日志

1. `Hello` 是否发出
2. `Discover` / `GetCapabilities` 是否响应
3. `AssignSlot` 是否成功 ACK
4. `Heartbeat` 是否稳定到达
5. `inventory_revision` 是否按预期变化
