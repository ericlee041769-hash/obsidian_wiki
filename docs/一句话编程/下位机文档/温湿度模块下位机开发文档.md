# 1. 文档目标

本文档用于定义首期 `温湿度模块` 的下位机开发范围、功能要求、协议行为、状态机、测试项和验收标准，作为固件开发、联调和测试的直接依据。

本文档默认继承主文档 [AI一句话编程](../AI一句话编程.md) 中的整体架构、自动编号、心跳机制和 CANFD 协议 v1 约束。

# 2. 模块定位

温湿度模块属于首期优先实现的低风险环境感知模组，用于：

1. 读取环境温度和相对湿度。
2. 为提醒、显示、联动控制和简单自动化提供基础输入。
3. 支持阈值订阅，在超温、低温、超湿或过干时主动上报事件。
4. 作为教学场景中的典型双通道传感器模组。

该模块本身不直接驱动高风险执行器，但它经常被作为规则判断输入，因此要求采样稳定、单位统一、阈值语义清楚。

# 3. 首期实现范围

## 3.1 必须实现

1. 模组上电 `Hello` 自报。
2. 自动编号与重编号。
3. 心跳保活。
4. `Discover` 和 `GetCapabilities`。
5. `read.temperature`。
6. `read.humidity`。
7. `read.status`。
8. `configure.sample_interval_ms`。
9. `configure.temp_high_threshold`。
10. `configure.temp_low_threshold`。
11. `configure.humidity_high_threshold`。
12. `configure.humidity_low_threshold`。
13. `configure.threshold_hysteresis`。
14. `subscribe.temperature_threshold`。
15. `subscribe.humidity_threshold`。
16. 本地参数持久化。
17. 两个按钮调整 `preferred_slot`。

## 3.2 首期可选实现

1. 露点计算。
2. 传感器自加热补偿。
3. 板载显示屏。
4. OTA。

## 3.3 首期不做

1. 多探头级联。
2. 本地长期历史存储。
3. 本地复杂环境补偿模型。
4. 空气质量复合估算。

# 4. 硬件假设

本文档基于以下硬件假设编写，若后续选型不同，需要同步修正文档和参数范围。

## 4.1 模块组成

1. MCU 一颗。
2. CANFD 收发器一颗。
3. 温湿度传感器一颗。
4. I2C 或单总线接口一组。
5. 状态 LED 一颗。
6. `slot_up` / `slot_down` 按钮各一个。
7. 非易失存储区，用于保存参数和地址偏好。

## 4.2 电气假设

1. 模块逻辑电压为 `3.3V`。
2. 传感器输出温度和湿度两个测量值。
3. 首期默认测量频率不高于 `1 Hz` 到 `2 Hz`。
4. 传感器自身采样时间小于配置的采样周期。

## 4.3 地址与身份

1. 每个模块出厂必须烧录唯一 `module_uid`。
2. 首次上电默认 `assigned_slot = 0xFF`。
3. `preferred_slot` 默认为 `1`，真正生效的地址由主机分配。

# 5. 功能需求

## 5.1 读取功能

模块必须提供以下读取能力：

1. `read.temperature`
2. `read.humidity`
3. `read.status`

说明：

1. `read.temperature` 返回当前环境温度，单位为摄氏度。
2. `read.humidity` 返回当前相对湿度，单位为百分比。
3. `read.status` 返回当前模块状态位图。

## 5.2 配置功能

模块必须支持以下配置能力：

1. `configure.sample_interval_ms`
2. `configure.temp_high_threshold`
3. `configure.temp_low_threshold`
4. `configure.humidity_high_threshold`
5. `configure.humidity_low_threshold`
6. `configure.threshold_hysteresis`

## 5.3 事件功能

模块必须支持：

1. `subscribe.temperature_threshold`
2. `subscribe.humidity_threshold`
3. 温度越上限、越下限与恢复事件上报
4. 湿度越上限、越下限与恢复事件上报

## 5.4 地址管理功能

模块必须支持：

1. 未分配状态接入。
2. 接收主机下发的 `assigned_slot`。
3. 保存 `preferred_slot`。
4. 按钮调整 `preferred_slot` 并发起改号请求。
5. 心跳中携带当前地址信息和最近采样摘要。

# 7. 数据定义

## 7.1 温度

1. 单位统一为摄氏度 `C`。
2. 编码类型使用 `Float`。
3. 首期建议保留一位或两位小数精度。

## 7.2 湿度

1. 单位统一为相对湿度百分比 `%RH`。
2. 编码类型使用 `Float`。
3. 取值范围为 `0 ~ 100`。

## 7.3 状态位建议

`read.status` 建议返回位图：

| Bit | 含义 |
| ---- | ---- |
| `bit0` | 在线且已分配地址 |
| `bit1` | 传感器初始化成功 |
| `bit2` | 最近一次采样成功 |
| `bit3` | 温度阈值告警中 |
| `bit4` | 湿度阈值告警中 |
| `bit5` | 最近一次配置写入成功 |
| `bit6` | 地址等待重新分配 |

# 9. 协议行为

## 9.1 上电流程

模块上电后必须按以下顺序执行：

1. 初始化时钟、接口、传感器、CANFD、定时器和存储。
2. 读取本地持久化参数。
3. 若无 `module_uid`，则判定为制造错误，不进入正常运行。
4. 完成一次初始采样或标记传感器未就绪。
5. 发送 `Hello`。
6. 等待主机 `Discover`、`GetCapabilities` 和地址配置。

## 9.2 Hello Payload

建议 `Hello` 至少包含：

```yaml
module_uid: 2A-11-9C-00-00-00-31
module_type: temp_humidity
preferred_slot: 1
assigned_slot: 255
address_state: unassigned
fw_version: 1.0.0
protocol_version: 1
sensor_model: sht30
```

## 9.3 Read 响应

### `read.temperature`

返回：

1. 编码类型 `Float`
2. 当前温度值

### `read.humidity`

返回：

1. 编码类型 `Float`
2. 当前湿度值

### `read.status`

返回：

1. 编码类型 `BitMap`
2. 当前状态位图

## 9.4 Configure 写入

模块收到 `Configure` 时必须：

1. 检查子功能码是否合法。
2. 检查参数类型是否合法。
3. 检查阈值与回差是否越界。
4. 检查高阈值不低于低阈值。
5. 更新运行时配置。
6. 对持久化参数做异步写入。
7. 返回 ACK 或错误码。

## 9.5 Subscribe 行为

模块收到订阅后：

1. 保存阈值事件订阅状态。
2. 在采样任务中判断阈值越界与恢复。
3. 仅在状态变化时主动上报。

# 10. 自动编号与按钮改号

## 10.1 地址模型

1. 同类型模块内 `assigned_slot` 唯一。
2. `0xFF` 表示未分配。
3. 主机是唯一地址分配者。
4. 模块只提出 `preferred_slot`，不能自行宣布地址生效。

## 10.2 按钮行为

### `slot_up`

1. 短按：`preferred_slot + 1`
2. 达到最大值后首期建议停留在最大值

### `slot_down`

1. 短按：`preferred_slot - 1`
2. 到达最小值后停留在 `1`

## 10.3 按钮改号后的动作

1. 更新本地 `preferred_slot`
2. 发送 `Event.SlotChangeRequest`
3. 等待主机下发新的 `assigned_slot`
4. 收到确认后更新本地状态

# 11. 心跳与在线状态

## 11.1 心跳内容

建议心跳至少包含：

```yaml
module_uid: 2A-11-9C-00-00-00-31
module_type: temp_humidity
assigned_slot: 1
address_state: assigned
latest_temperature: 25.3
latest_humidity: 48.0
status_flags: 0
uptime_sec: 180
```

## 11.2 心跳规则

1. 默认每 `10 秒` 发送一次。
2. 若主机要求，也可配置为 `30 秒` 一次。
3. 心跳发送失败时可记录错误计数，但不允许自行改地址。

# 12. 采样与滤波设计

## 12.1 采样任务

建议采用独立周期任务：

1. 每 `sample_interval_ms` 执行一次采样。
2. 连续读取传感器数据。
3. 对异常读取做重试或丢弃。
4. 生成当前温度和湿度结果。

## 12.2 平滑策略

首期建议采用轻量平均策略：

1. 连续保留最近 `3` 次有效数据。
2. 对外返回滑动平均结果。
3. 传感器异常时不使用无效值污染结果。

## 12.3 异常值处理

1. 温度或湿度超物理范围时判定为异常读数。
2. 异常读数不参与阈值判断。
3. 连续异常时更新状态位并继续心跳。

# 13. 阈值事件设计

## 13.1 温度阈值

1. 高于 `temp_high_threshold` 时触发高温事件。
2. 低于 `temp_high_threshold - threshold_hysteresis` 时上报恢复。
3. 低于 `temp_low_threshold` 时触发低温事件。
4. 高于 `temp_low_threshold + threshold_hysteresis` 时上报恢复。

## 13.2 湿度阈值

1. 高于 `humidity_high_threshold` 时触发高湿事件。
2. 低于 `humidity_high_threshold - threshold_hysteresis` 时上报恢复。
3. 低于 `humidity_low_threshold` 时触发低湿事件。
4. 高于 `humidity_low_threshold + threshold_hysteresis` 时上报恢复。

## 13.3 为什么要有回差

设置回差是为了避免阈值附近抖动导致频繁重复上报。

# 14. 固件状态机

建议固件至少包含以下状态：

1. `BOOT`
2. `UNASSIGNED`
3. `ASSIGN_PENDING`
4. `ASSIGNED_IDLE`
5. `MEASURING`
6. `FAULT`
7. `OTA`

## 14.1 状态说明

### `BOOT`

初始化硬件、读取参数、准备上报。

### `UNASSIGNED`

尚未获得主机分配的逻辑号。

### `ASSIGN_PENDING`

已经收到地址分配，等待确认写入和切换。

### `ASSIGNED_IDLE`

已分配地址，等待采样或协议请求。

### `MEASURING`

正在执行周期采样与阈值判断。

### `FAULT`

存在严重错误，例如传感器初始化失败、存储损坏或阈值配置非法。

# 15. 本地持久化内容

模块建议持久化以下参数：

1. `module_uid`
2. `preferred_slot`
3. 上次 `assigned_slot`
4. `sample_interval_ms`
5. `temp_high_threshold`
6. `temp_low_threshold`
7. `humidity_high_threshold`
8. `humidity_low_threshold`
9. `threshold_hysteresis`

说明：

1. 采样值本身不要求掉电保存。
2. 配置写入 Flash 时要做磨损控制。

# 16. 错误处理

## 16.1 推荐内部错误码

| 错误码 | 含义 |
| ---- | ---- |
| `0x01` | 传感器初始化失败 |
| `0x02` | 采样异常 |
| `0x03` | 阈值参数非法 |
| `0x04` | 参数越界 |
| `0x05` | Flash 写入失败 |
| `0x06` | 地址未分配 |

## 16.2 异常处理原则

1. 参数非法时拒绝写入。
2. 关键初始化失败时进入 `FAULT`。
3. 采样失败时继续发送心跳，便于主机定位问题。
4. 不允许因为一次采样失败就复位整机。

# 18. 测试项

## 18.1 功能测试

1. 上电后是否能正确发送 `Hello`
2. 主机分号后是否能进入 `Assigned`
3. `read.temperature` 是否返回有效值
4. `read.humidity` 是否返回有效值
5. 配置写入后是否立即生效
6. 掉电重启后参数是否保留

## 18.2 阈值测试

1. 超过高温阈值时是否主动上报
2. 低于低温阈值时是否主动上报
3. 超过高湿阈值时是否主动上报
4. 低于低湿阈值时是否主动上报
5. 恢复到安全区间后是否主动上报恢复
6. 阈值附近抖动时是否避免频繁重复事件

## 18.3 地址与心跳测试

1. 首次插入是否自动分号
2. 同类型两个模块同时插入是否能正确区分
3. 按钮改号后是否能正确触发重分发
4. 心跳是否按设定周期发送
5. 心跳超时后主机是否能判定离线

## 18.4 异常测试

1. 阈值上下限颠倒时是否拒绝生效
2. 采样异常时状态位是否正确
3. Flash 写入失败时是否能返回错误

# 19. 验收标准

满足以下条件即可认为首期温湿度模块下位机开发完成：

1. 能完成 `Hello -> Discover -> GetCapabilities -> AssignSlot -> Heartbeat` 全流程。
2. 能稳定返回 `read.temperature` 和 `read.humidity`。
3. 能正确保存和加载采样及阈值参数。
4. 能在阈值穿越时主动上报事件。
5. 连续运行 `24 小时` 无异常死机或协议失联。
